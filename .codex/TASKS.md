# 자주 쓰는 작업 프롬프트 (복사해서 Codex에 지시)

## 1) 코드베이스 설명(온보딩)

설명해줘:

- 레포 목적/아키텍처 개요(Next.js App Router 여부 포함)
- 라우팅 구조, 데이터 페칭 전략(SSR/SSG/ISR/CSR)
- 공통 UI 컴포넌트/유틸 정리
- 빌드/테스트/배포 파이프라인 요약(배포 스크립트 수정은 금지)

## 2) 접근성 개선 스윕

다음을 기준으로 a11y 점검하고 작은 PR 여러 개로 제안해줘:

- 시맨틱 태그/aria-label/role/대체텍스트
- 키보드 포커스 순서/스킵 링크
- 폼 레이블/에러 메시지 연결
- 콘트라스트/Focusable 컨트롤

## 3) 성능 최적화(초기)

LCP/CLS/JS 번들 기준으로 병목 찾고:

- 이미지/폰트 최적화, dynamic import
- React.memo/useMemo/useCallback 적용 후보
- 서버 컴포넌트 전환 가능 지점 제안
- Lighthouse 전/후 수치 캡처

## 4) 타입/테스트 보강

- any 제거 후보 목록과 대체 타입 제안
- 핵심 컴포넌트 테스트 템플릿 추가(RTL)
- e2e 스모크(홈 → 주요 링크 → 404 미방문) Playwright 초안

## 5) 버그 픽스 라운드

- 린트 에러/경고 정리
- 실행 시 경고/오류 재현 → 최소 수정 PR
- 예외/경계상태(빈 데이터/오류 fetch) UI 보강

## 6) 문서화

- README에 개발/빌드/테스트/릴리즈 절차 보강
- ADR(의사결정 기록) 초안: 라우팅/데이터전략/스타일가이드
